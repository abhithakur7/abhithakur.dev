## Overview

Rocket Rebates aggregates rebate data from third-party feeds and exposes it as a searchable catalog. I designed the data layer and ingestion pipeline, and built the NestJS APIs the product sits on.

## What I built

- **PostgreSQL schema on AWS RDS.** Designed the relational schema for large-scale rebate datasets: normalised core entities, staging tables for ingest, and compound indexes tuned to the query shapes the product actually runs.
- **ETL pipelines.** Ingest, normalise, and process third-party feeds at scale — handling dirty inputs, field mismatches, and partial re-ingests without double-writing.
- **REST APIs with NestJS.** Query planning and indexing strategies tuned for the filter combinations the UI exposes; pagination and sort paths designed around the indexes, not fitted around them after.
- **Full-text search layer.** Implemented for rebate discovery and filtering so users can find relevant offers by keyword alongside structured filters.

## Key decisions

- **Normalised source-of-truth + search index.** Kept the relational store clean and moved discovery/ranking load to the search layer; neither storage paid for the other's query patterns.
- **Idempotent ingestion.** Content-hashing inbound records so partial re-runs of the pipeline don't duplicate rebates or silently drop updates.
- **Indexes first, then API contracts.** Shaped query shapes around what Postgres could answer cheaply, rather than designing pretty APIs and hoping the indexes would materialise later.
