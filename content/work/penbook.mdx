## Overview

Penbook is a collaborative writing product where AI isn't a sidebar — it's woven into the drafting and editing loop. I built the backend on NestJS + MongoDB with OpenAI-driven generation and a real-time collaboration layer.

## What I built

- **NestJS + MongoDB backend.** Optimised query patterns for document-heavy read paths: selective projections, compound indexes on the access patterns the editor actually uses, and aggregation pipelines for workspace summaries instead of N+1 fetches.
- **OpenAI integration.** Powers AI-driven content creation and editing workflows inside the editor — prompt construction, request shaping, and response handling live server-side so the client never holds an API key.
- **Real-time collaboration.** Developed the collaborative editing primitives: presence, concurrent edits, and document sync so multiple writers can work on the same piece without clobbering each other.

## Key decisions

- **Server-side AI mediation.** Sits between the client and the model so prompts, context, and usage are all controlled from one place — easier to iterate on prompt design and to cap spend per workspace.
- **Mongo over SQL for the document store.** The editor's unit of work is a document with nested structure; modelling it relationally would have meant constant joins or JSON-in-a-column. Mongo fit the access pattern natively.
- **Event-driven sync loop.** Real-time edits propagate through a gateway channel per document, not a single global pipe — keeps fanout bounded and presence cheap.
